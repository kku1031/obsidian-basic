### 개념 
-  src
 └─ main
     ├─ java/               ← 소스 (Source) : 개발자가 작성한 코드 
     │    └─ com/kanggyeonggu/store/DemoApplication.java
     └─ resources/          ← 리소스 (Resource) : 코드가 아닌, 코드가 사용하는 외부 자원(파일)
          ├─ static/        ← 정적 파일(CSS, JS, 이미지 등) : 실행 중에 내용이 바뀌지 않는 파일
          ├─ templates/     ← 동적 템플릿(Thymeleaf, html 등)
          └─ application.properties

- 소스 : 컴파일 단계 ,자바 컴파일러가 `.class`로 변환
- 리소스 : Run 단계, 그대로 복사되어 배포 (가공 안 함)

## 14: 30 ~ 15 : 20
###  MVC의 의미

`MVC`는 소프트웨어 설계 패턴 중 하나로, **Model-View-Controller**의 약자입니다.

- **Model**: 데이터와 비즈니스 로직을 담당
    
- **View**: 사용자에게 보여주는 화면(UI)을 담당
    
- **Controller**: 사용자의 요청을 받아서 처리하고, Model과 View를 연결
		- 모델과 뷰의 교집합. 
		-
즉, Spring MVC는 **웹 애플리케이션에서 MVC 패턴을 적용한 프레임워크**라고 보면 됩니다.
